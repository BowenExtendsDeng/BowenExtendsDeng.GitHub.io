二十六，网络编程

1，基本概念：
（1）ip地址：

​ 用于唯一标识网络中的每一台计算机

​ 查看 ip 地址：ipconfig

​ ip 地址的表示形式：点分十进制

​ 每一个十进制数字的范围：0 ~ 255

​ 组成：网络地址 + 主机地址

（2）域名：

​ 为了方便记忆，解决 ip 记忆困难

​ 将 ip 地址映射成域名

（3）端口号：

​ 用于标识计算机上某个特定的网络程序

​ 表示形式：以整数形式，范围 0 ~65535

​ 0 ~ 1034 已经被占用，比如 ssh 22， ftp 21，smtp 25，hrrp 80

​ 常见的网络程序端口号：tomcat 8080，mysql 3306，oracle 1521，sqlserver 1433.

（4）TCP/IP协议：

​ 网络通讯协议，这个协议是 Intenet 最基本的协议，是网络层 IP 和传输层 TCP 协议组成的。

TCP协议（传输数据协议） UDP协议（用户数据协议）
1，使用 TCP 协议前，需建立 TCP 连接，形成传输数据通道 1，将数据、源、目的封装成数据包，不需要建立连接
2，传输前，采用 三次握手 的方式，是最可靠的 2，每个数据大小限制在 64k 以内
3，TCP 协议进行通信的两个应用进程：客户端，服务端 3，因无需连接，故是不可靠的
4，在连接中可进行大数据量的传输 4，发送数据结束时无需释放资源，速度快
5，传输完毕，需要释放已建立的连接，效率低 5，举例：发短信

​ 当客户端连接到服务器后，实际上客户端也是通过一个端口和服务器进行通讯的，这个端口是 TCP/IP 来分配的

​ Socket 允许程序把网络当成一个流，数据在两个 Socket 间通过 IO 传输

​ 一般主动发起通信的应用程序属于客户端，等待通讯请求的为服务端

5，UDP网络编程
（1）基本介绍：

​ 类 DatagramSocket 和 DatagramPacket （数据包）实现了基于 UDP 协议的网络程序。

​ UDP数据包通过数据包套接字 DatagramSocket 发送和接收，系统不保证 UDP 数据一定能够安全送到目的地，也不能确定什么时候可以抵达。

​ DatagramPacket 对象封装了 UDP 数据包，在数据包中包含了发送端的 IP 地址和端口号以及接收端的 IP 地址和端口号。

​ UDP 协议中每个数据包都给出了完整的地址信息，因此无需建立发送方和接收方的连接。

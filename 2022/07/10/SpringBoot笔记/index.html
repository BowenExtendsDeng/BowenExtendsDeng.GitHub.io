    二，原理简述

    • springbooot 在启动的时候，从类路径下 /META-INF/spring.factories 获取指定的值
    • 将这些自动配置的类导入容器，自动配置就会生效，帮我们进行自动配置
    • 以前我们需要自动配置的东西，现在springboot帮我们做了
    • 整合JavaEE，解决方案和自动配置都在spring-boot-autoconfigure-2.2.0.RELEASE.jar这个包下
    • 他会把所有需要导入的组件，以类名形式返回，这些组件就会被添加到容器中
    • 容器中也会存在非常多的xxxAutoConfiguration文件，就是这些类给容器中导入了场景需要的所有组件

    三，主启动类运行方法

    • 推断应用是普通项目还是Web项目

    • 找出所有的应用程序监听器，设置到 steners 属性中
    • 推断并设置 main 方法的定义类，找到运行的主类

    十，Swagger

    1，前后端分离：

    Vue + SpringBoot

    • 后端：后端控制层，服务层，数据访问层
    • 前端：前端控制层，视图层
      • 伪造后端数据：json
    • 前后端如何交互：调用API
    • 前后端相对独立，松耦合
    • 前后端可以部署在不同的服务器上

    问题：

    • 前后端集成联调，前端人员后后端人员不能做到及时协商，需高效沟通尽早解决

    解决方案：

    • 制定 schema，实时更新 API，降低集成的风险
    • 早些年：制定 Word 计划文档
    • 前后端分离：
      • 前后端测试接口：Postman
      • 后端提供接口，需要实时更新的消息及改动

    2，Swagger：

    • 流行的 API 框架
    • RestFul Api 文档在线自动生成工具=》Api文档与Api定义同步更新
    • 直接运行，可以在线测试Api接口
    • 支持多种语言

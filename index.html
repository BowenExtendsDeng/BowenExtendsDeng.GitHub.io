二十六，网络编程

1，基本概念：
（1）ip地址：

​ 用于唯一标识网络中的每一台计算机

​ 查看 ip 地址：ipconfig

​ ip 地址的表示形式：点分十进制

​ 每一个十进制数字的范围：0 ~ 255

​ 组成：网络地址 + 主机地址

（2）域名：

​ 为了方便记忆，解决 ip 记忆困难

​ 将 ip 地址映射成域名

（3）端口号：

​ 用于标识计算机上某个特定的网络程序

​ 表示形式：以整数形式，范围 0 ~65535

​ 0 ~ 1034 已经被占用，比如 ssh 22， ftp 21，smtp 25，hrrp 80

​ 常见的网络程序端口号：tomcat 8080，mysql 3306，oracle 1521，sqlserver 1433.

（4）TCP/IP协议：

​ 网络通讯协议，这个协议是 Intenet 最基本的协议，是网络层 IP 和传输层 TCP 协议组成的。

TCP协议（传输数据协议） UDP协议（用户数据协议）
1，使用 TCP 协议前，需建立 TCP 连接，形成传输数据通道 1，将数据、源、目的封装成数据包，不需要建立连接
2，传输前，采用 三次握手 的方式，是最可靠的 2，每个数据大小限制在 64k 以内
3，TCP 协议进行通信的两个应用进程：客户端，服务端 3，因无需连接，故是不可靠的
4，在连接中可进行大数据量的传输 4，发送数据结束时无需释放资源，速度快
5，传输完毕，需要释放已建立的连接，效率低 5，举例：发短信

​ 当客户端连接到服务器后，实际上客户端也是通过一个端口和服务器进行通讯的，这个端口是 TCP/IP 来分配的

线程

1，定义

​ 由进程创建，是进程的一个实体

​ 一个进程可以有多个线程

​ 单线程一个时刻一个进程只允许运行一个线程

​ 多线程指同一时刻一个进程可以运行多个线程

​ 并发：同一时刻，多个任务交替执行，造成一种貌似同时的错觉，简单地说，单核CPU实现的多任务就是并发

​ 并行：同一个时刻，多个任务同时进行，多核CPU可以实现并行

7，线程同步机制

（1）在多线程编程中，一些敏感数据不允许被多个线程同时访问，此时是哟个同步访问技术，可保证数据在任何时刻，最多有一个线程可以访问，以保证数据的完整性。

（2）另一种理解方式：线程同步，即当有一个线程在对内存进行操作时，其他线程都不可以对这个内存地址进行操作，直到该线程完成操作，其他线程才能对该内存地址进行操作。

（3）实现同步的方法

​ 同步代码块

​ 放在方法声明，表示整个方法为同步的方法

（4）互斥锁

​ JAVA 中引入互斥锁的概念，来保证共享数据操作的完整性。

​ 每个对象都对应于一个可称为“互斥锁”的标记，这个标记用来保证在任意时刻，只能有一个线程访问该对象。

​ 关键字 synchronized 来与对象的互斥锁联系。当某个对象用 synchronized 修饰时，表明该对象在任一时刻只能由一个线程访问。

​ 同步会导致程序执行效率降低。同步方法的锁可以是 this，也可以是其他对象。静态的同步方法锁是当前类本身。

​ 同步对象如果没有 static 修饰，默认对象为 this

​ 要求多线程的锁对象为同一个

（5）线程死锁

​ 多个线程占用了对方的资源，但不肯相让，导致了死锁，这是在编程中要尽力避免的。

（6）释放锁

​ 下面操作会释放锁

​ 当前线程在同步代码块，同步方法中出现了未处理的 Error 或者 Exception，导致异常结束

​ 当前线程在同步代码块，同步方法中执行了线程对象 wait() 方法，当前线程暂停，并释放锁

（四）StirngBuilder类

是StringBuffer一个简易替换，提供与StringBuffer兼容的API，但不保证同步（不是线程安全）。用于字符串缓冲区被单线程使用，如果可能，优先采用该类，因为比StringBuffer更快。StringBuilder主要是append和insert方法，可重载这些方法，以接受任意类型的数据。

2，特点：

（2）是final类

（3）对象的字符序列仍然存放在父类 AbstractStringBuilder 中

3，三者比较：

（1）String：不可变字符序列，效率低，复用率高

（2）StringBuffer：可变字符序列，效率高，线程安全

（3）StringBuilder：可变字符序列，效率最高，线程不安全

反射

1，介绍
（1）开闭原则：

​ 不修改源码的前提下扩展功能

（2）反射机制：

​ Classs 加载类，返回 Class 类型对象；通过加载到的类创建对象实例；通过加载到的方法创建方法实例；调用方法。

​ 反射机制允许程序在执行期间借助于 %Reflection API 取得任何类的内部信息（比如成员变量，构造器，成员方法等等），并能操作对象的属性及方法。发射在射击模式和框架的底层会用到。

​ 加载完类之后，在队中就产生了一个 Class 类型的对象（一个类只有一个 Class 对象），这个对象包含了类的完整信息。通过这个对象得到类的结构。这个 Class 对象就像一面镜子，透过这个镜子看到类的结构，所以称之为反射。

（3）举例：
（4）JAVA程序3个阶段：

​ 代码阶段/编译阶段，Class类加载阶段，Runtime运行阶段。

​ 编译产生字节码文件，字节码文件通过类加载器生成类对象/堆，之后运行，运行时 Class 对象同时知道自己属于的加载类。

（5）应用：

​ 在运行时判断任意一个对象所属的类

​ 在运行时构造任意一个类的对象

​ 在运行时得到任意一个类所具有的成员变量和方法

​ 在运行时调用任意一个对象的成员变量和方法

​ 生成动态代理

（6）主要的相关类：
类名 功能
java.lang.Class 代表一个类，Class对象表示某个类加载后在堆中的对象
java.lang.reflect.Method 代表类的方法
java.lang.reflect.Field 代表类的成员变量
java.lang.reflect.Constructor 代表类的构造方法

（6）优缺点：

优点：

​ 可以动态的创建和使用对象（也是框架底层核心），使用灵活，没有反射机制，框架技术就是去底层支持。

缺点：

​ 使用反射基本是解释执行，对执行速度又影响。

（1）介绍：

​ Class 也是类，因此也继承自 object 类

​ Class 类对象不是 new 出来的，而是系统创建的

​ 对于某个类的 Class 类对象，在内存中只有一份，因为类只加载一次

​ 每个类的实例都会记得自己是有哪个 Class 实例所生成

​ 通过 Class 可以完整的得到一个类的完整结构，通过一系列 API

​ Class 对象是存放在堆的

​ 类的字节码为二进制数据，是放在区的，有的地方成为类的元数据（包括代码，变量名，方法名，访问权限等等）

4，类加载
（1）类型：

​ 静态加载：

​ 编译时加载相关的类，如果没有则报错，依赖性太强

​ 动态加载：

​ 运行时加载需要的类i，如果运行时不用该类，则不报错，降低了依赖性

（2）源码编译运行流程：

​ JAVA源码通过**javac编译获得字节码文件，通过java进行加载**到 JVM 虚拟机中最终得到可执行的Java程序。

​ 加载操作首先是 加载（loading），之后进行 **连接（linking）**操作。

​ 连接步骤是依次进行 **验证（Verification），准备（Preparation），解析（Resolution），之后进行初始化**操作。

​ 类加载完后会有两个区域：方法区，堆区。

​ 方法区存放类的字节码二进制数据 ；堆区存放类的 Class 对象。

​ 堆区是访问数据的入口，方法区存放数据和访问权限等信息。

（3）加载操作：

​ 加载：将编译完成的 class 文件读入内存，并为之创建一个 java.lang.Class 对象，此过程由类的加载器完成

​ 连接：将类的二进制数据合并到 JRE 中

​ 验证：对文件安全性进行验证

​ 准备：对静态变量进行初始化并分配内存空间

​ 解析：转 符号引用 为 字节引用

​ 初始化：JVM负责对类进行初始化，这里主要是指静态成员

（4）加载：

​ JVM在该阶段的主要目的是将字节码从不同的的数据源（可能是 class 文件，也可能是 jar 包，甚至网络）转化为二进制的字节流加载到内存中，并生成一个代表该类的 java.lang.Class 对象

（6）验证：

​ 目的是为了确保 Class 文件的字节流中包含的信息符合当前虚拟机的要求，并且不会危害虚拟机自身的安全。

​ 包括：文件格式的验证（是否以魔数 oxcafebabe开头），元数据验证，字节码验证和符号引用认证

​ 可以考虑使用 -Xverify：none 参数来关闭大部分的类验证措施，缩短虚拟机类加载的时间

（7）准备：

​ JVM会在该阶段对静态（static）变量分配内存并初始化（对应数据类型的默认初始值，如 0，0L，null，false 等）。这些变量所使用的内存都将在方法区中进行分配

（8）解析：

​ 虚拟机将常量池内的符号引用替换为直接引用的过程

（9）初始化：

​ 真正开始执行类中定义的 Java 程序代码，此阶段是执行 () 方法的过程

​ () 方法是编译器按语句在源文件中出现的顺序，依次自动收集类中的所有静态变量的赋值动作和静态代码块的语句，并进行合并。

​ 虚拟机会保证一个类的 () 方法在多线程环境中被正确的加锁，同步，如果多个线程同时去初始化一个类，那么只会有一个线程去执行这个类的 () 方法，其他线程都需要阻塞等待，直到活动线程执行 () 方法完毕
